9. 后续维护与进阶

  • 每次写新文章：hexo new "文章标题" → 编辑 → hexo g -d（生成并部署）。
  • 修改主题配置：主题的配置文件在 themes/next/_config.yml（不同主题位置可能不同），可以自定义菜单、样式、评论系统等。
  • 添加分类和标签：在文章 Front-matter 中添加 categories 和 tags，然后在主题配置中启用对应菜单。
  • 添加自定义域名：如果你想用自己的域名，在 source 目录下创建一个 CNAME 文件（无后缀），内容为你的域名，然后在域名提供商处解析到 GitHub Pages 的 IP。
  • 备份源文件：博客的源文件（myblog 文件夹）可以单独建一个私有仓库保存，避免丢失。

常见问题

Q: 执行 hexo deploy 时报错 fatal: not a git repository？
A: 确保已经在博客根目录下初始化过 Git，或者检查 _config.yml 中的 repo 地址是否正确。

Q: 部署后页面样式错乱？
A: 通常是因为主题的静态资源路径配置问题。如果是自定义域名，需要检查 _config.yml 中的 url 是否设置正确。

Q: 本地预览正常，但部署后文章不显示？
A: 执行 hexo clean 后重新生成并部署。
